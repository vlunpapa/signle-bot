# 信号发送目标群组配置

## ✅ 配置完成

信号Bot的分析结果现在将发送到配置的目标群组（通过 `SIGNAL_CHAT_ID` 环境变量设置）

## 配置说明

### 环境变量
在`.env`文件中添加：
```env
SIGNAL_CHAT_ID=-1001234567890  # 示例：替换为你的实际群组ID
```

**注意**：Telegram超级群组的ID格式为 `-100` + 原始ID。如果配置的是正数ID，系统会自动尝试转换。

### 工作原理

1. **消息接收**：信号Bot在处理群（`RELAY_TARGET_CHAT_ID`）接收转发的消息
2. **Token分析**：提取Token并执行策略分析
3. **信号发送**：分析结果发送到目标群组（`SIGNAL_CHAT_ID`）

### 架构流程

```
处理群（`RELAY_TARGET_CHAT_ID`）
  ↓
信号Bot接收消息
  ↓
提取Token并分析
  ↓
执行策略（5分钟交易量告警等）
  ↓
发送交易信号到目标群组（`SIGNAL_CHAT_ID`）
```

## ⚠️ 重要提示

1. **确保信号Bot已加入目标群组**（`SIGNAL_CHAT_ID`）
   - 如果Bot未加入群组，发送会失败（403错误）

2. **验证配置**
   - 查看日志，应该看到：`信号将发送到目标群组: <你的群组ID>`

3. **测试**
   - 等待新消息触发策略
   - 检查目标群组是否收到交易信号

## 修改配置

如果需要更改目标群组，修改`.env`文件中的`SIGNAL_CHAT_ID`值，然后重启信号Bot。

## 日志示例

启动时应该看到：
```
信号将发送到目标群组: <你的群组ID>
```

发送信号时应该看到：
```
发送信号通知: xxx, 策略=交易信号1, 强度=XX, 目标群组=<你的群组ID>
信号已发送: 5分钟交易量告警 - xxx
```
